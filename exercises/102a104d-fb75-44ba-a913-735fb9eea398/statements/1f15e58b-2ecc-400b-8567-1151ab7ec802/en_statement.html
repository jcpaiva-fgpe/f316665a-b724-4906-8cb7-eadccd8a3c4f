Perfect Weddings

It is Saint Anthony's day in the realm of expression, and parentheses and braces are thrilled to be getting married. However, they still need find your match. The bride and groom will await their brides in order and, so as not to create too much confusion, they will leave as soon as they arrive. Like are modern people, they will even resort to a computer program to control this process.

Task

The program must read a line of text with an expression, which can contain curved brackets, straight brackets and braces. If all pairs are well matched, the program should print the type of each pair - () , [] or {} - and their respective opening and closing positions (starting at 0). A printout must be printed line for each pair, in the closing order (the second number is strictly growing). If any of the couples are ill-married, the program should only print a line saying Malformed pairs . If the line does not does not contain parentheses or braces, the program must print a line say No boyfriends to marry .

There is no need to do any validation of the expression other than marriage brackets and braces. The expression has no more than 1000 symbols. At output, the integers are separated from the pair type by a space, but the parentheses or braces have no separators between them.

Example 1

Input 

3 (x + 2) -4 {x + [2y- (27-z) + 8w]} - 1

Output 

() 1 5
() 15 20
[] 11 24
{} 8 25

Example 2

Input 

2 (x + 3]

Output 

Malformed couples

Example 3

Input 

2 (x + 3) +2)

Output 

Malformed couples

Example 3

Input 

2x + 4y + 7

Output 

No grooms to marry
