Rounded price

With the sudden increase in the maximum VAT rate to 23.25%, announced for 2015, we run into some calculation problems. To get an estimate of the price with VAT, we can round to the nearest whole, for defect (largest integer not superior).
Now, to get the tax amount, we multiply the price by 6 and we divide by 100, when it is the minimum rate (6%). Therefore, for 23.25%, we can now multiply by 2325 and divide by 10,000 (quotient of entire division). With some care, the rest of the entire division even would allow you to get the exact error, but it doesn't matter.

Task

Write a program to calculate the price with VAT rounded by defect.

Input

A line with an integer that represents the base price (excluding VAT).

Output

A line with an integer that represents the price with VAT, followed by a space, and the phrase " (maybe default estimate) ", without quotes.

Example 1

Input 

100

Output 

123 (maybe default estimate)

Example 2

Input 

15

Output 

18 (maybe default estimate)

Example 3

Input 

3560

Output 

4387 (perhaps default estimate)
