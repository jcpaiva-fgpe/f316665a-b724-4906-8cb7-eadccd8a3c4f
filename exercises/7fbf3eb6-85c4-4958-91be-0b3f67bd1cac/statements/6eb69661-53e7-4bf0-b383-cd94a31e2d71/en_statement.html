Roman Numerals

Problem

Classical Roman civilization developed a numbering system that lasted for many centuries and is still used today in references particulars: `` D. Manuel II '', `` century. XXI '', `` year MCMXCVIII '', by example.

In Roman notation the letters I, V, X, L, C, D, M are used to represent quantities from 1 to 1000:

I 1
V 5
X 10
L 50
Ç 100
D 500
M 1000
The main difference between Roman and Arabic notation (which we use today) is that the values ​​of the symbols are not positional, but rather added or subtracted according to the order in which they occur. First, the repeated symbols are grouped and their value added, for example: III is , XX is , MMM is . Next, we analyze the groups of symbols in the right to left:
  • The rightmost group is always positive .
  • If a group is worth more in module than its neighbor on the right, the value has positive sign: XII is , LVIII is .
  • If the group is worth less in module than the neighbor on the right, its value has negative sign : IX is , XXM is .
  • No distinct symbol groups with equal values ​​can occur (correspond to invalid numerals like `` XVV '' or `` VIIIII '').
Thus, the Roman numeral MCMXCVIII is worth .

Assignment

Write a program that reads a Roman numeral and writes its value in decimal.

Input

The input is a Roman numeral in a single line, in characters uppercase, less than 20 characters in length. You can assume that the numeral is always valid and its value does not exceed 5000.

Output

The output must have the value of the Roman numeral in decimal (followed by line change).

Example 1

Input


CCCLXXIX

Output


379

Example 2

Input


CDXLVII

Output


447

Example 3

Input


MMMDCCCXLVIII

Output


3848
