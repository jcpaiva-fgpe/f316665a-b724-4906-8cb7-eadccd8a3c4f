Morse auctions

The auctioneer TopasLeilões recently inaugurated a system electronic to control the auctions it organizes. With the new system, buyers place their bids using a very electronic device simple, where there is only one button. By briefly pressing the button, the device emits a short electronic "beep", which represents the dot character, '.'; charging more for longer, the device beeps more long, which represents the dash character, '-'.

Using sequences of dots and dashes, buyers communicate to the system the value of your bids, using Morse code.

Task

Write a program that accepts the bid sequence from the console received by the system and write, also on the console, after each bid, the highest bid amount received to date and the number of bids valid. A bid is valid if the sequence of dots and dashes represents unambiguously a number, according to the Morse code table of the decimal digits: 
     1  ·----
     two  ··---
     3 ··· -
     4 ···· -
     5 ·····
     6 - ····
     7 - ···
     8 --- ··
     9 ---- ·
     0 -----

Input

The input is a sequence of lines containing only '.' (dot) and '-' (hyphen, dash, minus sign). Each line represents a bid, which it can be valid or invalid, as explained in the previous section. THE sequence ends when a line with five dashes arrives, representing the number zero. A file has a maximum of 100 lines and each line has at maximum 40 characters.

Output

The output is a sequence of lines, one line for each line of the input, excluding the termination line. Two separate numbers appear on each line by a space: the first represents the highest bid to the input line corresponding number and the second the number of valid bids, up to that line of input. If there are no valid bids up to that line, the value of the highest bid valid will be 0 up to that line.

Example 1

Input 

.---------
-----

Output 

10 1

Example 2

Input 

.---------
.-.-......
-----

Output 

10 1
10 1

Example 3

Input 

.---------
.-.-......
..........
-----

Output 

10 1
10 1
55 2

Example 4

Input 

. --- .--
. ---- .-
-----

Output 

0 0
0 0
