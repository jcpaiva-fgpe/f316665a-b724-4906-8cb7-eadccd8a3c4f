Game of Life

The Game of Life is not a "conventional" game. Do not exist players, neither can win or lose. And once the pieces are placed, seemingly simple rules determine everything that will happen to follow, with very complex behaviors, full of surprises!

This game is played on a checkered board. Each grid (or cell) may be dead or alive . We mark a cell live with the character '#' and a dead cell with the character '.' . In this sense, a starting board valid for the game of life would be for example: 

.........
.........
.... # ....
... ### ...
... #. # ...
.... # ....
.........
.........

Each cell has 8 neighboring cells (except the cells at the limit of the board), which are the adjacent squares in all directions (including diagonals). Given an initial configuration, the board will evolving with the application of rules. So, for each cell, we count the number of neighbors with living cells and what happens next depends of this number:

  • A dead cell with exactly 3 living neighbors is born , staying alive in the next step;
  • A living cell with 2 or 3 living neighbors survives , remaining live in the next step;
  • In all other cases, a cell dies (if it was alive), or remains dead.

Note that this rule is applied to all cells at the same time, that is, to determine what happens at a given step we have to check what happens to each cell before we actually do the changes that the rules determine. We need your help. Will you be able to simulate the Game of Life?

Task

Write a program that, given an initial configuration of the game of life, get the status of that same board after one number of steps.

Input

In the first line of input there are 3 whole numbers L , C and P (separated by spaces), indicating respectively the number of lines on the board, the number of columns and the number of steps to be use. You can assume that 1 ≤ L, C ≤ 100 and 1 ≤ P ≤ 250 ;.

Follow exactly L lines, each containing exactly C characters, indicating the initial state of the board ( '#' for a living cell, and '.' for a cell dead).

Output

The output consists of L lines, each with C characters, indicating the state of the board after P steps of application of the rules. The characters to be used are the same as the input .

Example of Input 

8 9 3
.........
.........
.... # ....
... ### ...
... #. # ...
.... # ....
.........
.........

Example of Output 

.........
.... # ....
.. ##. ## ..
.. ##. ## ..
.. ##. ## ..
.... # ....
.........
.........
