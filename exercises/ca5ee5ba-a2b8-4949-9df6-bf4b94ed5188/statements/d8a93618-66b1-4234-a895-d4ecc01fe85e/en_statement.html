iFind - the App to find the what do you want

To find the store of a certain type that is as close as possible possible from where we are we can use an app for our mobile phone. That is what Manuel and his classmates thought. So they decide to develop such an application for smart phone (aka smart-phone ).

As they are organized they decided to divide the program into several modules, each one being responsible for a part. In the end they would integrate them into a application. Manuel decided to implement the module to determine the location of the request type that is the shortest distance from the user. The coordinates of the location where the user is and the type of store that intended, as well as a description of existing locations. For each location, coordinates and type are given (restaurant, shopping center, bar, coffee, etc.). If there are multiple locations of the desired type at the same distance, it should indicate what happened first in the data list.

Task

Write the program to process the data and respond to the request. The coordinates of each location, relative to the same reference (orthonormal), are defined by a pair of integers. The location type is identified by a sequence of capital letters (maximum three).

Input

The first line contains the description of the user's current location and the type of store you want, the coordinates and the type being separated by a space (for example, 100 -160 SM ). In the second line, there is the number of existing locations (there will never be more than 200). One line follows for each location with the respective data: point coordinates (integers) and type of separated by a space (for example, 140 120 R , which indicates a restaurant). The type is always a sequence of capital letters (not more than three). There may be stores other than those indicated in the example. The coordinates belong to the range [-300,300].

Output

A line with the coordinates of the requested place, followed by the indication of the type and an integer that is the square of the distance that it is (for example, 220 -140 SM 14800 ). It will therefore not be necessary (nor should) calculate square roots . As mentioned, if there are two or more locations at the same distance, will respond whichever comes first in the data. If there is no store of the desired type you should print None (note the capital letter in the first letter).

Example 1

Input 

100 -160 SM
15
140 120 R
-60 160 C
200 -200 R
-160 180 CC
-100 -100 SM
220 -140 SM
-40 160 C
180 100 CC
60 40 SM
180 -20 R
120 220 SM
100 -180 R
180 -100 CC
200 0 SM
-120 0 R

Output 

220 -140 SM 14800

Example 2

Input 

100 -160 SM
0

Output 

None
