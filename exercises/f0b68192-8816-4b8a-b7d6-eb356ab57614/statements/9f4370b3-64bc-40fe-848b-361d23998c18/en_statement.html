Digital Preservation

We just ran into a serious problem with our files digital, still based on CD-ROM and some on floppy disks. Many files are corrupted and it is not possible to fully recover the your information. Fortunately, there are many copies of each this we think it will be possible to recover most of the files.

Task

The recovery strategy is simple: analyzing bit by bit the various copies of the same file we will take the most bit for each position frequent. In case of equal frequency for a given bit, the 0 value. For example, if we have 4 files with the following 6-bit strings the recovery result will be as indicated on the last line 

011000
010100
011001
001111

011000

To simplify your work, the files have been converted into a string 0 and 1 . The first line input will consist of N and M, where N is the number of "files" and M is the number of "bits" of each file. The next N lines are strings of M characters containing only the characters 0 or 1 . It is known that N does not exceed 30 and M does not exceeds 255.

Example 1

Input 

4 6
011000
010100
011001
001111

Output 

011000

Example 2

Input 

2 4
1100
1010

Output 

1000
