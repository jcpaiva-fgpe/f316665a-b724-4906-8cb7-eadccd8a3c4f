O strange case of the mutant symbols

Alice invented a code made up of sequences of symbols (zeros and ones), which she uses to exchange secret messages with Bernardo. THE Teresa Coscuvilheira, who likes to be aware of everything that happens and doesnâ€™t happen, she was very frustrated when she found herself unable to decipher the messages that Alice left on Bernardo's secretary. As revenge because of such brazenness, she decided to tamper with Alice's secret messages. Whenever she found a new note on the desk, she hurried to replace 1 's with 0 ' s and 0 's by 1 's, but not many for Bernardo and Alice no suspect.

Realizing this strange phenomenon of mutant symbols - few, but enough to make sense of the messages - Alice arranged a solution to allow Bernardo to correct, or at least, detect some cases of mutations. The solution is as follows. For example, for a message \ (s_1 \, s_2 \, s_3 \, s_4 \, s_5 \, s_6 \, s_7 \, s_8 \, s_9 \), consisting of a sequence of nine symbols ( 0 's and 1 's), Alice started adding seven new symbols \ (\, p_1 \, p_2 \, p_3 \, p_4 \, p_5 \, p_6 \, p_7 \). The calculation of the new symbols is done thus. The nine original symbols are grouped in a square matrix (of dimension \ (3 \ times3 \), in this example). Each of the lines is added a new symbol \ (p_i \), with value 0 or 1 , so that the total number of 1 's in a row is even. For the columns the strategy is the same. Each of the columns is added a new symbol \ (p_i \) with value 0 or 1 , for so that the total number of 1 's in each column is even.

\ (\ begin {array} {lll | l} s_1 & s_2 & s_3 & p_4 \\ s_4 & s_5 & s_6 & p_5 \\ s_7 & s_8 & s_9 & p_6 \\ \ hline p_1 & p_2 & p_3 & p_7 \\ \ end {array} \)

The message to be transmitted would then consist of 16 symbols grouped as follows \ (\, s_1 \, s_2 \, s_3 \, s_4 \, s_5 \, s_6 \, s_7 \, s_8 \, s_9 \, p_1 \, p_2 \, p_3 \, p_4 \, p_5 \, p_6 \, p_7 \), that is, first you have the original sequence and then the control sequence. If in the analysis of the message it is a parity error is detected in a single row and column, then the error it's located.

Task

Given a received message, encoded as described above, indicate:

  • if no errors were detected;
  • or if an error was detected, indicating the position where it occurred in message received;
  • or if more than one error was detected.

Input

A line consisting of an integer \ (n \), a space, and a string of \ ((n + 1) ^ 2 \) characters ( 0 's and 1 ' s) corresponding to the message received, being \ (2 \ leq n \ leq 50 \). note that \ ((n + 1) ^ 2 = n ^ 2 + 2n + 1 \) and the square matrix formed with the message received has \ (n + 1 \) rows and columns.

Output

A single line with one of the following messages (without quotes):
" OK " - if no errors were detected.
" ERROR IN k " - if an error was detected in the \ (k \) position, with \ (1 \ leq k \ leq (n + 1) ^ 2 \).
\ (k \) is the position in the received message where it was detected (and corrected) the error.
" ERRORS " - if more than one error has been detected.
