Communication networks are increasingly busy and applications require an increase in bandwidth. In order to optimize the use of networks and being able to transport as many bytes as possible on the same connections, network coding ( network) was developed coding ).

In network coding, data is not sent exactly as it is generated by the applications, but combinations of pieces of that data are sent. As long as the combinations of pieces received are independent, it is possible to recover the original data. An example of an operator for these combinations (a "combiner") is the XOR (\ (\ oplus \)), which matches or-exclusive .

The interesting thing about XOR is its logical property that that \ (p \ oplus p = 0 \). That is, when we apply the operation to the same element (value) we get zero. And when we operate with zero we get the given value, i.e., \ (p \ oplus 0 = p \). So if we have \ (p \) and \ (r \) we have:

Task

This problem asks you to put your mathematical minds to work to develop a program that gives a list of messages received, decode as much of the individual data as possible. The following combinations of data in the various messages received. As output they must be The individual data that has been successfully decoded (for example, alphabetical order). Each letter (always lowercase) represents a given different. The various messages are provided on separate lines, and each The message may contain a combination of several letters.

Input

In the first line there is an integer \ (n \) which is the number of messages received, with \ (2 \ leq n \ leq 10 \). Following are \ (n \) lines: each has a different sequence of lowercase letters representing the data sent in a message. In each message, the letters are distinct and, globally, are not more than 10 different letters are used.

Output

A single line with the list of data (single letters) that is managed to decode separated by commas, and a space after each comma, and alphabetically ordered. If you can't decode nothing should simply have the word error .
